Project: SSH Brute Force Detector

GitHub: https://github.com/RadiiYevstratov/ssh-bruteforce-detector

What I learned:

WSL (Windows Subsystem for Linux): I set up a Linux environment on Windows and worked with real `auth.log` files. This project helped me improve my Linux command-line skills and become more comfortable navigating and analyzing log files.

Python `sys` module: I learned how to execute Python programs from the terminal, work with command-line arguments, and improve my error handling by validating user input and handling exceptions.

Git & GitHub: I strengthened my Git workflow by committing changes regularly, managing the repository, and using GitHub to track the project's development.

Log parsing: I gained experience processing Linux authentication logs, extracting relevant information, and identifying failed SSH login attempts.

Python fundamentals: I improved my understanding of file handling, dictionaries, sorting, functions, and working with structured data to generate meaningful reports from log files.


Project: SSH Brute Force Detector (Version 2)

GitHub: https://github.com/RadiiYevstratov/ssh-bruteforce-detector

What I learned

Command-line interface (CLI): I expanded the application by accepting command-line arguments, allowing users to configure the analysis period and the minimum number of failed login attempts required to trigger a detection. This improved my understanding of building flexible command-line applications.

Time-based log analysis: I learned how to analyze events within a configurable time window. I worked with timestamps, compared dates and times, and grouped failed login attempts based on a specified period.

Data analysis and algorithms: I improved my ability to process large log files efficiently by organizing log entries per IP address, counting failed attempts, and generating reports based on user-defined conditions.

Input validation and error handling: I strengthened my input validation by checking command-line arguments, handling invalid values, and ensuring the application behaved correctly when encountering malformed log entries or incorrect user input.

Python programming: I deepened my understanding of dictionaries, lists, loops, functions, datetime operations, sorting, and writing modular, maintainable code.

Software development: This version taught me how to improve an existing project by adding new features instead of rewriting it from scratch. I learned the importance of designing software that is configurable, reusable, and easier to maintain.
